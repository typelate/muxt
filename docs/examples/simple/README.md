# Simple Example: Edit-in-Place Fruit Table

A complete `package main` application: a table of fruit you can edit one row at a time. It shows the core Muxt loop — template names are routes, a receiver supplies the data — and the edit-in-place hypermedia pattern with no hand-written JavaScript — htmx attributes do the swapping.

## Run it

```bash
go generate ./...
go run .
```

Then open [http://localhost:8080](http://localhost:8080). Set `PORT` to use a different port.

## Layout

| File | What it holds |
|------|---------------|
| `main.go` | `Backend` (the receiver) and its methods; `main` wires `TemplateRoutes(mux, backend)`. |
| `templates.go` | The `templates` variable, the `//go:generate` directive, and the hand-written types `Row` and `EditRow`. |
| `index.gohtml` | Route templates and the `view-row`/`edit-row` fragments. |
| `template_routes.go` | Generated by `muxt generate`. Not edited by hand. |
| `template_routes_test.go` | Tests for the generated handlers. |

## Routes

| Template name | Receiver method |
|---------------|-----------------|
| `GET /{$} List(ctx)` | `List(context.Context) []Row` |
| `GET /fruits/{id}/edit GetFormEditRow(id)` | `GetFormEditRow(int) (Row, error)` |
| `PATCH /fruits/{id} SubmitFormEditRow(id, form)` | `SubmitFormEditRow(int, EditRow) (Row, error)` |
| `GET /help` | none |

`GET /fruits/{id}/edit` swaps a row into an edit form; `PATCH /fruits/{id}` saves the change and swaps the row back. The `view-row` and `edit-row` fragments are the two states each row toggles between.

## Generation

`templates.go` carries the directive that produces `template_routes.go`:

```go
//go:generate go run github.com/typelate/muxt generate --use-receiver-type Backend --use-receiver-type-package github.com/typelate/muxt/docs/examples/simple
```

`--use-receiver-type Backend` is what gives the handlers their typed parameters (`id int`, `form EditRow`) instead of bare strings.
