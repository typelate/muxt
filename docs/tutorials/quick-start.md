# Quick Start

Build a small server-rendered site with Muxt. By the end you will have a running web server that serves HTML from templates, with routes generated by Muxt.

## Prerequisites

- Go 1.22 or later
- `muxt` installed: `go install github.com/typelate/muxt@latest`

## Step 1: Create the module

```bash
mkdir hello-muxt && cd hello-muxt
go mod init example.com/hello-muxt
```

## Step 2: Create main.go

```go
package main

import (
	"embed"
	"html/template"
	"log"
	"net/http"
)

//go:embed *.gohtml
var templateFiles embed.FS

//go:generate muxt generate
var templates = template.Must(template.ParseFS(templateFiles, "*.gohtml"))

func main() {
	mux := http.NewServeMux()
	TemplateRoutes(mux)
	log.Fatal(http.ListenAndServe(":8080", mux))
}
```

This is all the Go you need. The `//go:generate` comment is how you invoke Muxt. The `templates` variable must be package-level, because Muxt finds it via static analysis. From here, every new route is just a new template.

## Step 3: Create a template

Create `index.gohtml`:

```html
{{define "GET /"}}
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Hello</title></head>
<body>
  <h1>Hello from Muxt!</h1>
</body>
</html>
{{end}}
```

The template name `"GET /"` is the route. Muxt generates a handler for it that renders this template with the incoming `*http.Request` as data.

## Step 4: Generate and run

```bash
go generate && muxt check && go run .
```

Open [http://localhost:8080](http://localhost:8080). You should see "Hello from Muxt!".

`go generate` wrote `template_routes.go` containing a `TemplateRoutes` function that registers your route on the mux. You don't edit that file, just regenerate it whenever you change template names.

Take a look: search for `HandleFunc` in `template_routes.go`. There's one entry per template name, and the pattern is passed straight to the standard library's `ServeMux`. Muxt didn't add a router — it wired your template names into the one Go already ships.

## Step 5: Use the URL in your template

Muxt also passes the `*http.Request` as part of the template data. That means you can read path parameters and query strings directly in the template.

Add a greeting route to `index.gohtml`:

```html
{{define "GET /hello/{name}"}}
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Hello</title></head>
<body>
  <h1>Hello, {{.Request.PathValue "name"}}!</h1>
  {{with .Request.URL.Query.Get "from"}}
  <p>Sent from {{.}}.</p>
  {{end}}
</body>
</html>
{{end}}
```

`{name}` in the route pattern becomes a path parameter. `.Request.PathValue "name"` reads it back out. `.Request.URL.Query.Get "from"` reads an optional query string value; the `{{with}}` block skips the paragraph entirely when the parameter is absent.

Regenerate and run:

```bash
go generate && muxt check && go run .
```

Try these URLs:

- [http://localhost:8080/hello/world](http://localhost:8080/hello/world) renders "Hello, world!"
- [http://localhost:8080/hello/world?from=Go](http://localhost:8080/hello/world?from=Go) adds "Sent from Go."

No Go code changed, only the template. Muxt updated the route registration automatically.

While you're in `template_routes.go`, search for `ReadHelloByName`. Muxt generated a type-safe URL builder from the route pattern. You can use `{{$.Path.ReadHelloByName "world"}}` in any template instead of a hardcoded string — if you rename the route, the compiler tells you everywhere that reference breaks.

## Step 6: Make it interactive with htmz    

We could have used fixi, HTMX, or data-star for this example, but I thought it would be cool to highlight this nifty one-liner.

[htmz](https://leanrada.com/htmz/) is a tiny HTML snippet that lets you load server responses into any element on the page. Its entire source is one `<iframe>`:

```html
<iframe hidden name=htmz onload="setTimeout(()=>document.querySelector(contentWindow.location.hash||null)?.replaceWith(...contentDocument.body.childNodes))"></iframe>
```

When a form or link targets the `htmz` iframe, the iframe navigates to the URL. After it loads, htmz reads the URL's fragment (`#big-number`) to find the target element on the page, and replaces it with the response body's children. No application JavaScript needed.

Add a fragment route to `index.gohtml`. This template returns just a `<div>`, not a full page:

```html
{{define "GET /number"}}
<div id="big-number">
  <p style="font-size:8rem;text-align:center">{{.Request.URL.Query.Get "n"}}</p>
</div>
{{end}}
```

Then update `GET /` to add the form, the placeholder, and the htmz iframe:

```html
{{define "GET /"}}
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Hello</title></head>
<body>
  <h1>Hello from Muxt!</h1>

  <form method="get" action="/number#big-number" target="htmz">
    <input type="number" name="n" value="42">
    <button>Show</button>
  </form>

  <div id="big-number">
    <p style="font-size:8rem;text-align:center">?</p>
  </div>

  <iframe hidden name=htmz onload="setTimeout(()=>document.querySelector(contentWindow.location.hash||null)?.replaceWith(...contentDocument.body.childNodes))"></iframe>
</body>
</html>
{{end}}
```

The form's `action="/number#big-number"` tells htmz which element to replace. `target="htmz"` routes the submission through the iframe instead of navigating the page.

Regenerate and run:

```bash
go generate && muxt check && go run .
```

Open [http://localhost:8080](http://localhost:8080), type a number, and click Show. The `<div id="big-number">` on the page is replaced with the server's response. The Go code in `main.go` hasn't changed. The new route came from the new template name.

Search for `HandleFunc("GET /number"` in `template_routes.go` and compare it to `HandleFunc("GET /"`. They're structurally identical. Muxt has no concept of a fragment versus a full page — that distinction lives entirely in your template content.

## Step 7: Add a static route

Add one more template to `index.gohtml`:

```html
{{define "GET /about"}}
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>About</title></head>
<body>
  <h1>About</h1>
  <p><a href="/">Home</a></p>
</body>
</html>
{{end}}
```

Regenerate and run:

```bash
go generate && muxt check && go run .
```

Visit [http://localhost:8080/about](http://localhost:8080/about).

## What's next

The templates here use `*http.Request` as data because they have no method call. Once you want to fetch from a database or run real business logic, you add a receiver method and name it in the template. The [template name syntax reference](../reference/template-names.md) covers the full syntax including method calls and status codes. The [add-logging tutorial](add-logging.md) shows how to add structured logging to your generated handlers.
